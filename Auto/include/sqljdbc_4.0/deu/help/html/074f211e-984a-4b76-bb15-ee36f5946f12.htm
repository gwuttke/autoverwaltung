Neues in JDBC-Treiber

Microsoft JDBC Driver 4.0 für SQL Server enthält mehrere neue Funktionen und Erweiterungen.

Weitere Informationen zum Herstellen von Verbindungen mit einer SQL Azure-Datenbank

Ein neues Thema mit Informationen zum Herstellen von Verbindungen mit einer SQL Azure-Datenbank ist vorhanden. Weitere Informationen finden Sie unter Verbinden mit einer SQL Azure-Datenbank.

Unterstützung für hohe Verfügbarkeit, Notfallwiederherstellung

Microsoft JDBC Driver 4.0 für SQL Server unterstützt Notfallwiederherstellungsverbindungen für hohe Verfügbarkeit mit AlwaysOn-Verfügbarkeitsgruppen in SQL Server 2012. Weitere Informationen finden Sie unter JDBC Driver-Unterstützung für hohe Verfügbarkeit, Notfallwiederherstellung.

Herstellen von Verbindungen mit SQL Server mit der integrierten Kerberos-Authentifizierung

Microsoft JDBC Driver 4.0 für SQL Server bietet für Anwendungen die integrierte Kerberos-Authentifizierung Typ 4 zum Herstellen von Verbindungen mit einer SQL Server-Datenbank. Weitere Informationen finden Sie unter Herstellen von Verbindungen mit SQL Server mit der integrierten Kerberos-Authentifizierung. (Die integrierte Kerberos-Authentifizierung Typ 2 ist in Microsoft JDBC Driver für SQL Server-Versionen vor 4.0 verfügbar.)

Zugriff auf Diagnoseinformationen im erweiterten Ereignisprotokoll

Sie können auf Informationen im erweiterten Ereignisprotokoll des Servers zugreifen, um Verbindungsfehler zu untersuchen. Weitere Informationen finden Sie unter Zugriff auf Diagnoseinformationen im erweiterten Ereignisprotokoll.

Zusätzliche Unterstützung für Spalten mit geringer Dichte

Wenn die Anwendung bereits auf Daten in der Tabelle zugreift, die Spalten mit geringer Dichte verwendet, sollte sich die Leistung verbessern. Sie können Informationen über Spalten (einschließlich Informationen über Spalten mit geringer Dichte) mit getColumns-Methode (SQLServerDatabaseMetaData) abrufen. Weitere Informationen über Spalten mit geringer Dichte in SQL Server finden Sie unter Verwenden von Spalten mit geringer Dichte.

Die isSparseColumnSet-Methode wird der SQLServerResultSetMetaData-Klasse hinzugefügt, um anzugeben, ob eine Spalte in einem Resultset ein Satz von Spalten mit geringer Dichte ist.

Xid.getFormatId

In Versionen vor Microsoft JDBC Driver 4.0 für SQL Server hat der JDBC-Treiber eine Format-ID von der Anwendung empfangen, diesen Wert aber nicht an den Datenbankserver weitergegeben. Stattdessen hat der JDBC-Treiber die Format-ID 0 an den Datenbankserver weitergegeben.

Ab Microsoft JDBC Driver 4.0 für SQL Server gibt der JDBC-Treiber die Format-ID von der Anwendung an den Datenbankserver weiter. Um das aktualisierte Verhalten zu verwenden, stellen Sie sicher, dass sqljdbc_xa.dll auf dem Server aktualisiert wurde. Weitere Informationen zum Kopieren einer aktualisierten Version von sqljdbc_xa.dll auf den Server finden Sie unter Grundlegendes zu XA-Transaktionen.

Keine Itanium-Unterstützung für JDBC Driver 4.0-Anwendungen

Das Ausführen von Microsoft JDBC Driver 4.0 für SQL Server-Anwendungen auf einem Itanium-Computer wird nicht unterstützt.
