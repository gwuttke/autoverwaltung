SQLServerConnection-Elemente

Die folgenden Tabellen enthalten die Elemente, die von der SQLServerConnection-Klasse verfügbar gemacht werden.

Konstruktoren

Keine

Felder

Name

Beschreibung

TRANSACTION_SNAPSHOT

Dient zum Angeben der Transaktionsisolationsstufe für Momentaufnahmen.

Geerbte Felder

Klasse geerbt von:

Beschreibung

java.sql.Connection

TRANSACTION_NONE, TRANSACTION_READ_COMMITTED, TRANSACTION_READ_UNCOMMITTED, TRANSACTION_REPEATABLE_READ, TRANSACTION_SERIALIZABLE

Methoden

Name

Beschreibung

clearWarnings

Löscht alle für dieses SQLServerConnection-Objekt gemeldeten Warnungen.

close

Gibt die Datenbank für dieses SQLServerConnection-Objekt sowie die JDBC-Ressourcen umgehend frei, sodass nicht auf deren automatische Freigabe gewartet werden muss.

commit

Macht alle Änderungen, die seit dem letzten Commit oder Rollback vorgenommen wurden, zu dauerhaften Änderungen und hebt sämtliche Datenbanksperren auf, die derzeit von diesem SQLServerConnection-Objekt aufrecht erhalten werden.

createBlob

Erstellt ein java.sql.Blob-Objekt ohne Daten.

createClob

Erstellt ein java.sql.Clob-Objekt ohne Daten.

createNClob

Erstellt ein java.sql.NClob-Objekt ohne Daten.

createStatement

Erstellt ein SQLServerStatement-Objekt zum Senden von SQL-Anweisungen an die Datenbank.

createSQLXML

Erstellt ein java.sql.SQLXML-Objekt ohne Daten.

getAutoCommit

Ruft für dieses SQLServerConnection-Objekt den aktuellen Modus für automatische Commits ab.

getCatalog

Ruft den aktuellen Katalognamen für dieses SQLServerConnection-Objekt ab.

getClientConnectionID-Methode (SQLServerConnection)

Ruft die Verbindungs-ID des letzten Versuchs der Verbindungsherstellung ab, wobei es keine Rolle spielt, ob dieser Versuch erfolgreich war oder fehlgeschlagen ist.

getClientInfo

Ruft Informationen zu den Eigenschaften für Clientinformationen ab, die vom JDBC-Treiber unterstützt werden.

getHoldability

Ruft die aktuelle Haltbarkeit von SQLServerResultSet-Objekten ab, die unter Verwendung dieses SQLServerConnection-Objekts erstellt werden.

getMetaData

Ruft ein SQLServerDatabaseMetaData-Objekt mit Metadaten zu der Datenbank ab, für die dieses SQLServerConnection-Objekt eine Verbindung darstellt.

getTransactionIsolation

Ruft die aktuelle Transaktionsisolationsstufe für dieses SQLServerConnection-Objekt ab.

getTypeMap

Ruft das Map-Objekt ab, das diesem SQLServerConnection-Objekt zugeordnet ist.

getWarnings

Ruft die erste Warnung ab, die von Aufrufen für dieses SQLServerConnection-Objekt gemeldet wurde.

isClosed

Gibt an, ob dieses SQLServerConnection-Objekt geschlossen wurde.

isReadOnly

Gibt an, ob dieses SQLServerConnection-Objekt schreibgeschützt ist.

isValid

Gibt an, ob dieses SQLServerConnection-Objekt noch geöffnet und gültig ist.

nativeSQL

Konvertiert die angegebene SQL-Anweisung zur systemeigenen SQL-Grammatik des Datenbankservers.

prepareCall

Erstellt ein SQLServerCallableStatement-Objekt zum Aufrufen von in der Datenbank gespeicherten Prozeduren.

prepareStatement

Erstellt ein SQLServerPreparedStatement-Objekt zum Senden von parametrisierten SQL-Anweisungen an die Datenbank.

releaseSavepoint

Entfernt das angegebene SQLServerSavepoint-Objekt aus der aktuellen Transaktion.

rollback

Macht alle im Rahmen der aktuellen Transaktion vorgenommenen Änderungen rückgängig und hebt sämtliche Datenbanksperren auf, die derzeit von diesem SQLServerConnection-Objekt aufrecht erhalten werden.

setAutoCommit

Legt für dieses SQLServerConnection-Objekt den aktuellen Modus für automatische Commits auf den angegebenen Zustand fest.

setCatalog

Legt den angegebenen Katalognamen fest, um einen Teilbereich der Datenbank dieses SQLServerConnection-Objekts auszuwählen, in dem gearbeitet werden soll.

setClientInfo

Legt den Wert der Eigenschaften für Clientinformationen fest.

setHoldability

Ändert die Haltbarkeit von SQLServerResultSet-Objekten, die unter Verwendung dieses SQLServerSavepoint-Objekts erstellt werden, zur angegebenen Haltbarkeit.

setReadOnly

Versetzt dieses SQLServerConnection-Objekt in den schreibgeschützten Modus, damit vom JDBC-Treiber die Datenbankoptimierungen aktiviert werden.

setSavepoint

Erstellt in der aktuellen Transaktion einen unbenannten Sicherungspunkt und gibt das neue SQLServerSavepoint-Objekt zurück, das für den Sicherungspunkt steht.

setTransactionIsolation

Ändert die Transaktionsisolationsstufe für dieses SQLServerConnection-Objekt zur angegebenen Stufe.

setTypeMap

Installiert das angegebene TypeMap-Objekt als Typzuordnung für dieses SQLServerConnection-Objekt.
